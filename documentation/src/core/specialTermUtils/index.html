specialTermUtils.js Pure utilities

Three pure functions that bridge the ISpecialTerms adapter data and the planner's grid and coverage logic. These are the only functions that know how to interpret a SpecialTermDuration's weight field — all span and attribute-grant computations run through here.

Parent
core/
Consumed by
PlannerContext, planModel, Header, GradPanel, BankPanel, SemRow, SummerRow
Rule
No React, no I/O. Pure functions only.

Functions

resolveTermByDuration(durations, duration) → SpecialTermDuration | null
Finds the SpecialTermDuration descriptor that matches a stored duration value (in calendar months). Falls back to the first entry if no exact match is found — ensures the planner never crashes on edge-case plan data.

Why this is needed: The plan state only stores the integer month count (e.g. 4 or 6). The descriptor object is needed to get the weight for span arithmetic and the label for display. This function is the bridge between stored state and runtime descriptor.
Returns: The matching descriptor, or the first descriptor if no match, or null if durations is empty
// Usage — wherever a stored duration needs to become a descriptor
const type  = specialTerms.types.find(t => t.id === termData.typeId);
const descr = resolveTermByDuration(type.durations, termData.duration);
// descr.weight — for span logic
// descr.label  — for display
termSpans(termWeight, semSlotWeight) → boolean
Returns true if a term with the given grid weight overflows into the next consecutive semester slot.

Rule: termWeight > semSlotWeight. A term spans when its weight exceeds the weight of the slot it's placed in.

NU examples:
  • 6-month co-op (weight 2.0) in fall (weight 1.0) → 2.0 > 1.0 → spans into spring ✓
  • 4-month co-op (weight 1.0) in sumA (weight 0.5) → 1.0 > 0.5 → spans into sumB ✓
  • 4-month co-op (weight 1.0) in fall (weight 1.0) → 1.0 > 1.0 → false, single slot ✓
  • 2-month internship (weight 0.5) in sumA (weight 0.5) → 0.5 > 0.5 → false, single slot ✓
This replaces hardcoded if (duration === 6) and if (duration === 4 && summer) checks that existed before the refactor.
Returns: boolean — true if the term extends into the next semester
computeGrantedAttrs(specialTermPl, types) → Set<string>
Computes the set of attribute codes automatically granted by all currently placed special terms. Iterates every entry in specialTermPl that has a valid semId (is actually placed), finds the term type's descriptor, and unions the attributeGrants arrays.

Primary use case: The result is passed as grantedAttrs to attributeSystem.getCoverage(). This is how a placed co-op causes the EX (Integration Experience) NUPath cell to turn green in the grad panel — without any hardcoding of "EX" outside the NU adapter.

This was previously hardcoded as covered.add("EX") in 8+ places across the codebase whenever a co-op was present. The refactor centralized it here and moved the "EX" string into the adapter definition where it belongs.
Returns: Set<string> — all attribute codes granted by placed special terms. Empty set if none placed or no grants defined.
// Usage in GradPanel, Header, etc.
import { computeGrantedAttrs } from "../core/specialTermUtils.js";

const grantedAttrs = computeGrantedAttrs(specialTermPl, specialTerms?.types);
const npCovered    = attributeSystem.getCoverage(placements, courseMap, grantedAttrs);
