InstitutionContext.jsx Context

The bridge between the adapter bundle and the React component tree. Wrap the app root once with <InstitutionProvider>, then call usePort(IPortName) anywhere to read that port's implementation. 75 lines — thin by design.

Parent
context/
Adapter source
src/adapters/ via src/config.js
Exports
InstitutionProvider, useInstitution(), usePort(portKey)

Setup — already done in App.jsx

InstitutionProvider wraps the entire app tree above PlannerProvider, so adapter data is available to every component including PlannerContext itself.

// src/App.jsx
import { institutionAdapter } from "./config.js";
import { InstitutionProvider } from "./context/InstitutionContext.jsx";

function App() {
  return (
    <InstitutionProvider adapter={institutionAdapter}>
      <ThemeProvider>
        <PlannerProvider>
          ...
        </PlannerProvider>
      </ThemeProvider>
    </InstitutionProvider>
  );
}

Reading a port in a component

Import the port constant and pass it to usePort(). Port constants are camelCase strings — the same keys used in wire(). Importing from src/ports/ is what catches typos at the import site rather than silently at runtime.

// Most common pattern — read a single port
import { usePort }           from "../context/InstitutionContext.jsx";
import { IAttributeSystem }  from "../ports/IAttributeSystem.js";

function MyComponent() {
  const attributeSystem = usePort(IAttributeSystem);
  const covered = attributeSystem.getCoverage(placements, courseMap, grantedAttrs);
  // ...
}

usePort() vs useInstitution()

HookReturnsWhen to use
usePort(IPortName)One port implementationAlmost always — reads one port, minimizes what the component depends on
useInstitution()Whole adapter bundleOnly when you need multiple ports at once or must pass the bundle as a parameter to a non-React function (e.g. exportReport(…, adapter))

API reference

InstitutionProvider({ adapter, children })
Context provider component. Wraps the component tree once at the app root. adapter must be a fully-wired adapter bundle from wire() — i.e. the value of institutionAdapter from src/config.js.
usePort(portKey) → port implementation
Returns the implementation bound to portKey. Must be called inside a component tree wrapped by <InstitutionProvider> — throws otherwise. portKey should be a constant imported from src/ports/*.js.
useInstitution() → adapter bundle
Returns the entire wired adapter object (all 8 ports). Useful when passing to non-React utilities like exportReport() that need multiple ports. Prefer usePort() for component-level access.

How PlannerContext uses the adapter

PlannerContext calls useInstitution() once and destructures the ports it needs. This is the primary consumer of the adapter in the React layer — most UI components get institution data via PlannerContext (re-exposed as context values) rather than calling usePort() directly.

// Inside PlannerContext.jsx
const adapter = useInstitution();
const { attributeSystem, specialTerms, calendar, creditSystem } = adapter;
// These are then used in useMemo hooks and exposed in the context value

Components that need a port not exposed by PlannerContext — like GradPanel needing ISpecialTerms for computeGrantedAttrs — call usePort() directly.
