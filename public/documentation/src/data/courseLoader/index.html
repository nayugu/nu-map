Course Loading ICourseCatalog adapter

Course loading is the responsibility of the ICourseCatalog adapter. There is no standalone courseLoader.js — the adapter owns the full fetch-and-normalize pipeline and PlannerContext calls courseCatalog.fetchAll() directly on mount.

Contract
ICourseCatalog port — fetchAll(), normalize(), courseUrl()
NU implementation
src/adapters/northeastern/courseCatalog.js
Called by
PlannerContext.jsx — const courseCatalog = usePort(ICourseCatalog); courseCatalog.fetchAll()

How course loading works

On mount, PlannerContext reads the ICourseCatalog port from the active institution adapter and calls fetchAll(). The adapter decides how to source the data — static file, live API, or any other strategy. The result is already normalized to the Course shape before PlannerContext ever sees it.

// PlannerContext.jsx (simplified)
const courseCatalog = usePort(ICourseCatalog);

useEffect(() => {
  courseCatalog.fetchAll()
    .then(courses => {
      setCourses(courses);      // already normalized
    })
    .catch(err => setLoadError(err.message));
}, []);
ℹ
The adapter handles the entire strategy: local file first, API fallback, retry logic, error messages — all without touching PlannerContext. This is why there is no courseLoader.js wrapper. The adapter is the loader.

Northeastern implementation

The NU adapter (northeastern/courseCatalog.js) fetches a bundled static JSON first, then falls back to the live SearchNEU API if the local file is unavailable. All Banner term code decoding ("202430" → "spring") happens inside normalizeCourse() — PlannerContext never sees raw registrar codes.

// src/adapters/northeastern/courseCatalog.js (simplified)
export default {
  async fetchAll() {
    const json = await tryLocal() ?? await tryAPI();
    return json.map(normalizeCourse).filter(Boolean);
  },
  normalize: normalizeCourse,
  courseUrl: course => `https://catalog.northeastern.edu/course-descriptions/${course.subject.toLowerCase()}/`,
  getSources: () => [{ id: "nu-courses", label: "ninest/nu-courses", ... }],
};

Implementing for your institution

Override courseCatalog in your adapter folder. The minimum is a fetchAll() that returns Course[]. See ICourseCatalog for the full Course shape and the complete port contract.

Updating the NU course catalog

# Full update workflow
npm run data:scrape:write  # fetch from catalog.northeastern.edu (~20 min)
npm run data:patch:write   # apply YAML corrections from data/patches/
npm run build
git add public/northeastern/all-courses.json
git commit -m "data: update course catalog"
