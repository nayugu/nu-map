ports/ Interface contracts

Nine JSDoc-only files — no logic, no runtime behavior — that define exactly what data shape each institution adapter must provide. IAttributable is the base interface all ports extend. The remaining eight define institution-specific behavior.

Parent
src/
Implemented by
northeastern/ (NU), generic/ (fallback defaults)
Consumed via
usePort(IPortName) inside any component; useInstitution() for multi-port access
Rule
No logic, no imports. Pure JSDoc typedef + one named export for the port key string.

Base interface

base interface
IAttributable
All ports extend this. Defines getSources() → SourceInfo[]. wire().getAllSources() aggregates across all active ports to build the About modal attribution list.

The eight ports

identity
IInstitution
Name, short name, portal name, degree audit name, localStorage prefix. The minimum every fork must set.
semester structure
ICalendar
Semester types, weights, start year, year format. Weight is the foundation of all span arithmetic.
credit units
ICreditSystem
Unit name ("SH", "credits", "ECTS"), standard value, full-time min, semester max.
gen-ed / NUPath
IAttributeSystem
Attribute grid layout, labels, getCoverage(). Empty grid if institution has no attribute system.
work terms
ISpecialTerms
Co-op, internship, exchange — any non-course block that occupies semester slots. Empty if unused.
path helpers
IMajorRequirements
fmtLabel and fmtLocation — translate raw folder names into display labels for the major/minor selector.
data source
ICourseCatalog
fetchAll(), normalize(), courseUrl(). The adapter owns the full fetch-and-normalize pipeline — PlannerContext calls fetchAll() and receives Course[] already normalized.
UI copy
ILocalization
Disclaimer bullets in the About modal. Institution names, portal names, and legal language.

How ports are wired and consumed

flowchart LR P["src/ports/\nJSDoc contracts"] NU["adapters/northeastern/\nNU implementation"] GEN["adapters/generic/\nfallback defaults"] W["wire(overrides)\n{ ...generic, ...overrides }"] CFG["src/config.js\nexport institutionAdapter"] APP["App.jsx\nInstitutionProvider"] CTX["InstitutionContext\nusePort(IPortName)"] C1["any component\nusePort(IAttributeSystem)"] NU -->|"implements"| P GEN -->|"implements"| P NU --> W GEN --> W W --> CFG --> APP --> CTX --> C1
Port keys flow from contracts → implementations → wire() → InstitutionProvider → usePort(). Components only see the implementation, never the adapter folder.

Reading a port in a component

// Import the port key constant and the hook
import { usePort }          from "../context/InstitutionContext.jsx";
import { IAttributeSystem } from "../ports/IAttributeSystem.js";

function MyComponent() {
  const attributeSystem = usePort(IAttributeSystem);
  // attributeSystem is the full implementation from the active adapter
  const covered = attributeSystem.getCoverage(placements, courseMap, grantedAttrs);
}

Adding a new port

Only add a port when multiple components need the same institution-specific value and it doesn't fit an existing port. Four files to touch:

  1. Create src/ports/IMyPort.js — export the key string constant and a @typedef.
  2. Add a generic implementation in src/adapters/generic/myPort.js and wire it into src/adapters/generic/index.js.
  3. Add the key string to KNOWN_PORTS in src/adapters/wire.js.
  4. Add the optional field to AdapterOverrides in src/ports/index.js.

The app core and all existing components are untouched. The new port is available to any component via usePort(IMyPort).
