ICourseCatalog data source

The adapter owns the full course loading pipeline: fetch strategy, normalization, Banner term code decoding, and fallback logic. PlannerContext calls fetchAll() once on mount and receives Course[] already normalized — no raw catalog format ever reaches the core.

Port key
"courseCatalog"
Consumed by
PlannerContext (fetchAll on mount), InfoPanel (courseUrl for external link)
Override priority
Required — always implement fetchAll() to point at your institution's data
Also extends
IAttributable — must implement getSources()

Contract

MethodSignaturePurpose
fetchAll()() → Promise<Course[]>Fetch and normalize the full course catalog. The adapter chooses the strategy — static file, remote API, fallback chain, etc. Returns courses already in the Course shape. Called once by PlannerContext on mount.
normalize(raw)(raw: object) → Course|nullNormalize one raw catalog record into the Course shape. Returns null if the record lacks required identity fields. Exposed separately so individual records can be refreshed without re-fetching the full catalog.
courseUrl(course)(course: Course) → string|nullReturn the official course detail URL in the registrar catalog. Returns null if per-course URLs are not available. Used in InfoPanel for the external link icon. Optional — may be omitted.
getSources()() → SourceInfo[]External data sources this adapter draws from. See IAttributable. Used by wire().getAllSources() to build the About modal attribution list.

Course shape

Every normalized record must conform to this shape. Fields the core relies on:

FieldTypeNotes
idstringPrimary key — subject + number, e.g. "CS3500". Used in placements map, courseMap, etc.
subjectstringUppercase subject code, e.g. "CS", "MATH". Used for color assignment.
numberstringCourse number, e.g. "3500".
codestringHuman-readable combined code, e.g. "CS 3500".
titlestringFull course title.
shnumberCredit hours (minimum for variable-credit courses). Named sh for historical reasons; the actual unit name comes from ICreditSystem.
shMin / shMaxnumber|nullNull for fixed-credit courses. Non-null when the course has a credit range.
termsstring[]Semester type IDs when offered, e.g. ["fall", "spring"]. Already decoded from raw registrar codes by the adapter. Empty array means offering data is unavailable.
attributesstring[]Gen-ed / attribute codes, e.g. ["ND", "FQ"]. Codes defined by IAttributeSystem. Empty array if none.
prereqsPrereqNode[]Prerequisite expression tree (Major2 format). Empty array means no prerequisites.
coreqsCourseRef[]Flat corequisite list. Empty array means none.
colorstringDeterministic hex color derived from subject code via subjectColor() in courseModel.js.
✦
Attribute field name: Use attributes in new course data. The legacy field name nuPath is still accepted: raw.attributes ?? raw.nuPath ?? []. Terms must be decoded by the adapter — the core only sees semester type IDs like "fall", never raw Banner codes like "202430".

Northeastern reference implementation

// src/adapters/northeastern/courseCatalog.js
export default {
  async fetchAll() {
    // try local bundled JSON first, fall back to live API
    let json;
    try {
      json = await tryFetch(LOCAL_URL);
    } catch {
      json = await tryFetch(API_URL);  // SearchNEU API
    }
    return json.map(normalizeCourse).filter(Boolean);
  },

  normalize: normalizeCourse,  // decodes Banner terms, maps raw fields

  courseUrl(course) {
    return `https://catalog.northeastern.edu/course-descriptions/${course.subject.toLowerCase()}/`;
  },

  getSources() {
    return [{
      id: "nu-courses", label: "ninest/nu-courses",
      url: "https://github.com/ninest/nu-courses",
      author: "ninest", usedFor: "course catalog data",
    }];
  },
};

normalizeCourse() handles Banner term code decoding ("202430" → "spring"), maps the legacy nuPath field to attributes, and sanitizes description strings. All of this is an adapter concern — the core never sees raw ninest records.

Generic default

// src/adapters/generic/courseCatalog.js
export default {
  async fetchAll() {
    const res = await fetch(`${import.meta.env.BASE_URL}courses.json`);
    const raw = await res.json();
    return raw.map(normalizeCourse).filter(Boolean);
  },
  normalize: normalizeCourse,
  courseUrl() { return null; },
  getSources() { return []; },
};

Points to public/courses.json at the root. No live API fallback — if the file is missing, the app shows a fetch error.

Implementing for your institution

  1. Build a JSON array of courses. Minimum required fields: subject, number, title.
  2. Place it at public/<yourschool>/all-courses.json.
  3. Implement fetchAll(): fetch the file, map each record through your normalize() function, filter nulls.
  4. Implement normalize(raw): map your raw format to the Course shape. Decode any term codes to semester type IDs here.
  5. Optionally implement courseUrl(course) if your registrar has per-course URLs.
  6. Implement getSources(): return attribution info for the About modal (or return [] if you own the data).
