ICourseCatalog data source

Two URLs — that's the entire contract. LOCAL_URL points to your bundled course JSON; API_URL is an optional live fallback. Override both to point at your institution's data.

Port key
"courseCatalog"
Consumed by
courseLoader.js (passed as parameter by PlannerContext)
Override priority
Required — always set LOCAL_URL to your institution's course data path

Contract

FieldTypePurpose
LOCAL_URLstringPath to the bundled local course JSON in public/. Use import.meta.env.BASE_URL as a prefix so the path resolves correctly on any deploy base path. Fetched first on startup.
API_URLstring | nullLive API fallback endpoint. Fetched if the local JSON fetch fails (404 or network error). Set to null to disable the fallback — the app will show an error if the local fetch fails.

Northeastern reference implementation

// src/adapters/northeastern/courseCatalog.js
export const LOCAL_URL = `${import.meta.env.BASE_URL}northeastern/all-courses.json`;
export const API_URL   = "https://husker.vercel.app/courses/all";

export default { LOCAL_URL, API_URL };

The northeastern/ prefix in the local path namespaces NU's data under public/northeastern/, keeping the public folder clean for future institutions or generic files.

Generic default

// src/adapters/generic/courseCatalog.js
export const LOCAL_URL = `${import.meta.env.BASE_URL}courses.json`;
export const API_URL   = null;   // no live fallback by default

export default { LOCAL_URL, API_URL };

Expects courses.json at the public root. No API fallback — if the file is missing the app shows a fetch error.

Course JSON format

The JSON file must be an array of course objects. The fields consumed by normalizeCourse() in courseModel.js:

FieldTypeNotes
idstringUnique course identifier, e.g. "CS3500"
namestringFull course title
subjectstringSubject prefix, e.g. "CS", "MATH". Used for color assignment.
classIdstring | numberCourse number within the subject, e.g. "3500"
creditsnumberCredit hours. Falls back to creditSystem.standardValue if missing.
attributesstring[]Attribute/gen-ed codes, e.g. ["ND", "EI"]. Also accepts legacy nuPath field.
prereqsobject?Prerequisite expression tree (Major2 format). Optional.
coreqsobject?Corequisite expression tree (Major2 format). Optional.
descstring?Course description. Optional.
termsstring[]?Term codes indicating when the course is offered. Optional — used for offering confidence display.
✦
Attribute field name: Use attributes for new data. The legacy field name nuPath is still accepted via a compat shim in normalizeCourse(): attributes: raw.attributes ?? raw.nuPath ?? [].

Implementing for your institution

  1. Build or export a JSON array of courses in the format above.
  2. Place it at public/<yourschool>/all-courses.json (namespace under your school's folder).
  3. Set LOCAL_URL = `${import.meta.env.BASE_URL}<yourschool>/all-courses.json`.
  4. Set API_URL to a live endpoint if one exists, or null for local-only.
