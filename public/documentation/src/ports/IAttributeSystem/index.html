IAttributeSystem gen-ed / NUPath

"Attributes" is the generic name for what NU calls NUPath, and what other schools call gen-ed, distribution requirements, or learning outcomes. The grad panel grid, course chips, and PDF export all read from this port. Leave gridCodes empty and the grid disappears entirely.

Port key
"attributeSystem"
Consumed by
GradPanel (grid), Header (coverage badge), CourseCard (attribute chips), PDF export (labels), specialTermUtils (computeGrantedAttrs feeds here)
Override when
Your institution has a codified gen-ed or distribution requirement framework

Contract

FieldTypePurpose
systemNamestringDisplay name shown as the grad panel section title. e.g. "NUPath", "Gen Ed", "GIRs". Empty string → section is hidden.
attributesAttribute[]Full list of attribute definitions including legacy codes that exist in course data but are no longer active requirements.
gridLayoutstring[][]2-D array of attribute codes for the grad panel grid, e.g. [["ND","EI","IC","FQ"], ...]. Excludes legacy-only codes. Row × column order is display order.
gridCodesstring[]Flat ordered list from gridLayout.flat(). This is the source of truth for rendering and the coverage progress badge.
labelsObject<string,string>Code → label map for all attributes including legacy codes. Used for PDF export and tooltip lookups — must include every code that appears in course data.
getCoverage(placements, courseMap, grantedAttrs?)functionReturns Set<string> of covered attribute codes. See below.

getCoverage() signature

getCoverage(
  placements:   { [courseId: string]: semId: string },  // courses in the plan
  courseMap:    { [courseId: string]: Course },          // full course objects
  grantedAttrs: Set<string>                             // from computeGrantedAttrs()
): Set<string>

grantedAttrs is optional (defaults to empty Set). Compute it with computeGrantedAttrs(specialTermPl, types) before calling getCoverage. This is how a placed co-op causes the EX cell to turn green — no hardcoded "EX" in the attribute system itself.

// Standard usage in GradPanel, Header
import { computeGrantedAttrs } from "../core/specialTermUtils.js";

const grantedAttrs = computeGrantedAttrs(specialTermPl, specialTerms?.types);
const npCovered    = attributeSystem.getCoverage(placements, courseMap, grantedAttrs);

systemName: "NUPath". getCoverage iterates course.attributes for all placed courses, then adds every code from grantedAttrs.

Legacy code handling: WF appears in course data from before ~2018. It is in attributes and labels so PDF export resolves the label correctly, but it is not in gridLayout or gridCodes, so the grad panel never shows it as a requirement to satisfy.

Generic default

// src/adapters/generic/attributeSystem.js
export const systemName = "";   // no section title → grid hidden
export const attributes = [];
export const gridLayout = [];
export const gridCodes  = [];
export const labels     = {};
export function getCoverage() { return new Set(); }

With an empty gridCodes, the GradPanel hides the attribute grid entirely. No errors, no empty boxes.

Implementing for your institution

  1. Define your attribute codes as { code, label } objects in the attributes array.
  2. Build gridLayout as a 2-D array — each inner array is one row of the grid. Arrange codes in the display order you want.
  3. Derive gridCodes = gridLayout.flat().
  4. Derive labels = Object.fromEntries(attributes.map(a => [a.code, a.label])). Add any legacy codes manually if they appear in your course data.
  5. Implement getCoverage: iterate placed courses, union their attributes arrays, add grantedAttrs.
  6. Set systemName to your framework's display name.
✦
Course objects use the generic field name attributes (not nuPath). Your course JSON should include an attributes array with the short codes, e.g. "attributes": ["ND", "FQ"]. The normalizeCourse() function in courseModel.js reads raw.attributes ?? raw.nuPath for backwards compatibility.
