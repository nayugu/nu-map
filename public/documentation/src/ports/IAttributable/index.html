IAttributable base interface

Every port adapter must implement getSources(). Adapters that draw from external data return one SourceInfo per upstream source. Adapters that own their own data return []. wire().getAllSources() aggregates across all active ports and deduplicates by id — the About modal calls this one method to build its complete attribution list.

SourceInfo shape

FieldTypeRequiredPurpose
idstringYesStable unique key for deduplication. Must be the same across all ports that reference the same upstream source — e.g. both courseCatalog and a hypothetical future port could both claim "nu-courses" and getAllSources() would merge them. Never change this value after deployment.
labelstringYesRepository or dataset name as shown in the About modal. Typically "org/repo" format, e.g. "sandboxnu/graduatenu".
urlstringYesCanonical URL — GitHub repo, dataset page, or project site.
authorstringNoGitHub username or org. Shown as @username next to the label. Omit if attribution is already clear from the label.
usedForstringYesPlain-English description of what this source provides, shown as "used for ___" in the About modal. E.g. "graduation requirement definitions", "course catalog data".

How getAllSources() works

wire(overrides) adds a getAllSources() method to the merged adapter bundle. It iterates all active port implementations, calls getSources() on each, and deduplicates by id. When two ports return the same id, their usedFor descriptions are merged into an array so the About modal shows a single source entry with multiple uses.

// Simplified wire.js logic
merged.getAllSources = () => {
  const map = new Map();
  Object.values(merged)
    .filter(p => typeof p?.getSources === "function")
    .flatMap(p => p.getSources())
    .forEach(s => {
      if (map.has(s.id)) {
        const existing = map.get(s.id);
        map.set(s.id, { ...existing, usedFor: [existing.usedFor, s.usedFor].flat() });
      } else {
        map.set(s.id, { ...s, usedFor: [s.usedFor] });
      }
    });
  return [...map.values()];
};

Adding getSources() to an adapter

// In any adapter file that uses an external data source:
export default {
  // ... other port methods ...
  getSources() {
    return [{
      id:      "my-source",
      label:   "myorg/myrepo",
      url:     "https://github.com/myorg/myrepo",
      author:  "myorg",
      usedFor: "course catalog data",
    }];
  },
};

If the adapter owns its own data and has no external upstream: getSources() { return []; }.
