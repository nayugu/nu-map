calendar.js — semester weights

NU's academic year uses four semester types. Their weights are the foundation of all span arithmetic — any duration with weight > semesterSlotWeight spills into the next slot.

idlabelweightoptionalmonths
fallFall1.0falseSep–Dec
springSpring1.0falseJan–May
sumASummer 10.5trueMay–Jun
sumBSummer 20.5trueJul–Aug

Year total: 1.0 + 1.0 + 0.5 + 0.5 = 3.0. calendar.js also exports Stage 1 compat shims (DEFAULT_START_YEAR, SEMESTER_TYPES) for backwards compatibility with code that imports from constants.js.

specialTerms.js — co-op and internship

The two term types reflect NU's co-op program and optional internship policy. The key insight is the weight vs. duration distinction: duration is the calendar length stored in the plan; weight drives the span logic in the grid.

index.js — wiring

Imports all 8 adapter files and re-exports them as a typed overrides object. This is what wire() in src/config.js receives.

// src/adapters/northeastern/index.js
import nuInstitution       from "./institution.js";
import nuCalendar          from "./calendar.js";
// ... 6 more imports

/** @type {import('../../ports/index.js').AdapterOverrides} */
const overrides = {
  institution:       nuInstitution,
  calendar:          nuCalendar,
  // ... 6 more ports
};
export default overrides;
