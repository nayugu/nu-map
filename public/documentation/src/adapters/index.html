adapters/ Institution implementations

The adapters layer contains every institution-specific value and rule in the entire codebase. Swap the import in src/config.js and the planner reconfigures for a different school — semester structure, credit units, gen-ed framework, work-term types, course catalog URL, disclaimers, and all.

wire() — adapter composition

wire(overrides) composes a complete adapter bundle by merging institution-specific overrides on top of the generic defaults. Any port not listed in overrides keeps its generic implementation. In development, it warns on unknown port keys.

// src/adapters/wire.js (simplified)
import genericDefaults from "./generic/index.js";

export function wire(overrides = {}) {
  if (import.meta.env.DEV) {
    // warn on unknown port keys to catch typos early
    for (const key of Object.keys(overrides)) {
      if (!KNOWN_PORTS.has(key)) console.warn(`[wire] Unknown port: "${key}"`);
    }
  }
  return { ...genericDefaults, ...overrides };
}
wire(overrides?) → adapter bundle
Merges overrides on top of generic defaults. Returns a plain object with all 8 port implementations. Called once at build time in src/config.js.

config.js — the single file to change

src/config.js is the only file outside src/adapters/ that knows which institution is active. Everything else reads through usePort().

// src/config.js
import { wire }              from './adapters/wire.js';
import northeasternOverrides from './adapters/northeastern/index.js';

export const institutionAdapter = wire(northeasternOverrides);

To switch institutions: change the import on line 2 to point at a different adapter folder. Nothing else needs to change.

Forking for a new institution — minimum steps

✦
Minimum viable fork: Override only institution (identity) and courseCatalog (data URL). Everything else uses generic defaults and the planner will function with no NUPath grid, no special term types, and "SH" as the credit unit.
  1. Copy src/adapters/northeastern/ to src/adapters/myuniversity/
  2. Edit institution.js — set id, name, shortName, storagePrefix
  3. Edit courseCatalog.js — set LOCAL_URL to your bundled course JSON path
  4. Edit other files as needed: calendar.js for semester structure, specialTerms.js for work-term types, attributeSystem.js for gen-ed, etc.
  5. Edit src/config.js — change the import to your new folder
  6. Place your course data JSON at the path specified in courseCatalog.js

Generic defaults

The generic adapter provides safe fallbacks for every port. Institutions should override these rather than rely on them in production, but they make the planner functional with minimal configuration:

PortGeneric default behavior
institutionid "map", name "My University", prefix "map"
calendarFall + Spring only, weight 1.0 each, year starts September
creditSystemunitName "SH", standard 4, full-time min 12
attributeSystemEmpty system — no grid, getCoverage() returns empty Set
specialTermsNo term types — empty types array
majorRequirementsIdentity functions — no label transformation
courseCatalogPoints to public/all-courses.json with no live fallback
localizationSingle generic disclaimer bullet
