generic/ Fallback defaults

Safe, working defaults for every port. A fork that only overrides institution and courseCatalog gets a functional planner — no attribute grid, no work terms, but a working course browser, major selector, and plan builder.

Minimum viable fork

Two overrides give you a working planner for any institution:

// src/adapters/myuniversity/index.js
import { wire } from "../wire.js";
import myInstitution  from "./institution.js";
import myCourseCatalog from "./courseCatalog.js";

export default wire({
  institution:  myInstitution,    // required: name, prefix, etc.
  courseCatalog: myCourseCatalog, // required: path to your courses.json
  // everything else uses generic defaults
});

With only these two overrides: course browser works, plan building works, major/minor loading works (if your data folder matches the expected layout). The grad panel attribute grid is hidden. No co-op/work-term blocks in BankPanel. Credit unit label shows as "credits".

wire() — how overrides compose

// src/adapters/wire.js (simplified)
import genericDefaults from "./generic/index.js";

export function wire(overrides = {}) {
  if (import.meta.env.DEV) {
    // warn on unknown port keys to catch typos early
    for (const key of Object.keys(overrides)) {
      if (!KNOWN_PORTS.has(key)) console.warn(`[wire] Unknown port: "${key}"`);
    }
  }
  return { ...genericDefaults, ...overrides };
}

It's a simple spread. Generic defaults are the base; your overrides replace individual ports. No deep merge — each port is replaced atomically. If you override calendar, you replace the entire calendar implementation, not individual fields.

Forking checklist

  1. Copy src/adapters/northeastern/ to src/adapters/myuniversity/
  2. Required: Edit institution.js — set id, name, shortName, portalName, degreeAuditName, storagePrefix
  3. Required: Edit courseCatalog.js — set LOCAL_URL to your data path; set API_URL or null
  4. Required: Place your course data JSON at the path set in step 3
  5. Optional: Edit calendar.js if your semester structure differs from Fall/Spring/SumA/SumB
  6. Optional: Edit creditSystem.js if your unit name or credit values differ
  7. Optional: Edit attributeSystem.js to define your gen-ed codes and grid
  8. Optional: Edit specialTerms.js to define your work-term types
  9. Optional: Edit localization.js to insert your institution names in disclaimers
  10. Edit src/config.js — change the import from northeastern/ to myuniversity/

Named exports for selective import

generic/index.js provides both a default bundle export and named port exports so you can import individual sub-adapters:

// Import only what you need
import { calendar, creditSystem } from "../generic/index.js";

// Use a generic sub-adapter as a base and override specific fields
export default wire({
  institution: myInstitution,
  calendar: {
    ...calendar,             // spread generic calendar as base
    defaultStartYear: 2025,  // override just this one field
  },
});
